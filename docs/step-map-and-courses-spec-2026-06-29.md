# ステップ一覧（冒険マップ）＋ コース制 仕様 — 2026-06-29

子ども側「ステップをえらぼう」画面を、楽しくデザイン性の高い **冒険マップ（空に登るすごろく道）** に作り直す。
あわせて、ステップ群を束ねる単位として **コース（学年別 / 英検級別）** を導入する。

> ステータス: **デザイン確定・仕様ドラフト（未実装）**。動くプロトタイプあり（`docs/spike/StepMapPrototype.swift`）。
> 関連: [eiken-level-mapping.md](./eiken-level-mapping.md) / [ai-ocr-handwriting-research-2026-06-27.md](./ai-ocr-handwriting-research-2026-06-27.md) / コンテンツ管理スキーマv2。

---

## 1. 背景・ゴール

現状の子ども側ステップ選択は「横並びのシンプルなカード（アイコン円＋タイトル＋単語数＋状態）」。
機能はするが遊び心が薄い。今後ステップが大量に増える前提で、

- **ステップが何個増えても破綻しない**見せ方
- **進んだ実感（達成感・物語性）**が出る見せ方
- 学年や英検級で**中身が切り替わる**構造

を満たす「空に登る冒険マップ」に刷新する。

## 2. UI/UX 設計思想との整合（重要）

CLAUDE.md の2ユーザー軸（子＝やる人／親＝管理する人）に従う。

- 子に **級・レベル・採点で優劣を意識させない**。→ コース名は子にはやさしい呼称で見せる（例: 「小1のたび」「きいろのたび」）。英検◯級という級ラベルを子画面に直接出さない。
- 子画面は **1画面1メイン動作**: 光っている「いまここ」を押すだけ。
- 複雑さ（コース設定・許可・採点）は **親ゲートの奥**に隠す。
- 既存の見た目言語（角丸・パステル・カプセルバッジ・バウンス/Haptic）を再利用し、新しい見た目を不用意に増やさない。

## 3. 画面構成（子ども側）

### 3.1 レイアウト
- **縦スクロール**。**下＝スタート → 上にスクロールするとそのコースの世界を登っていく**。
  - 背景は**コースのテーマ（§3.5 WorldTheme）で変わる**。高く登った＝進んだ、を色と飾りで体感。
  - ステップが増えても「上に伸ばすだけ」で破綻しない。最上部に `テーマのゴール絵文字＋つづく…`。
- **縦・横どちらの向きでも**使える（スクロール式）。横向きは道幅が広がるので間隔チューニング前提（§7）。
- 起動時は **「いまここ」へ自動スクロール**（`ScrollViewReader.scrollTo(anchor: .center)`）。

### 3.2 上部バー（常時固定）
- **おうちボタン 🏠（左上）**: ホームに戻る。子が必ず迷子にならない出口を用意。
- タイトル「ステップをえらぼう」（中央・既存トーン）。
- **コースチップ（右上）**: 現在のコースを `絵文字＋やさしい名前` で表示。既定は**表示のみ（切り替え不可）**。§5 参照。

### 3.3 ノード（ステップ1個）の状態
| 状態 | 見た目 | 補足 |
|---|---|---|
| できた (done) | 緑の大チェック＋スタンプ風リング＋「できた！」バッジ | 鍵マークは出さない（いつでも押せる） |
| いまここ (current) | 白縁＋コース色の塗り＋★＋**拍動リング**＋「いまここ」＋「◯こ」 | 足元に**じぶんのアバター 🧒**（avatar-dressup 連携） |
| これから (upcoming) | 破線の淡いノード＋✨＋「◯こ」 | ロックではなく「やさしく誘う」表現 |

- 道中に **なかまキャラ**（nakama-character-gen 連携）を点在させる。
- 「優劣を見せない」ため、これから=暗い/鍵ではなく**淡く誘う**トーンにする。

### 3.4 活動種別（ActivityKind）を「形」で区別
ノードの **形** で活動の種類を表す（色・形の語彙を増やしすぎない）。
| 種別 | 形 | アイコン | 用途 |
|---|---|---|---|
| 単語練習 (word) | 丸 | ★ / ✨ | 通常のスペル練習ステップ |
| カメラ取り込み (camera) | 角丸四角（オレンジ系） | 📷「しゃしんをとる」「カメラ」 | 手書きをカメラで撮ってAI採点（AI-OCR） |

将来 ことばパズル等の別 ActivityKind を足す場合も、この「形で種別」の方針を踏襲する。

### 3.5 各学年で「世界」を変える（WorldTheme）（重要）
1学年ぶんはステップが大量＝スクロールが長くなる。**全コースが同じ「宇宙に登るだけ」の共通UIだと飽きる**ため、
**コースごとに世界観（テーマ）を丸ごと差し替える**。背景グラデ・地面色・小道色・点在する飾り・ゴール絵文字をテーマで持つ。

- データモデル（プロトタイプ実装済み）:
  ```
  WorldTheme {
    skyStops: [Gradient.Stop]   // 空のグラデ（上→下）
    ground: Color               // 地面（スタート側の大地）
    path:   Color               // 小道の色
    goalEmoji, goalLabel        // 最上部「つづく…」の絵柄
    props:  [Prop]              // 点在する飾り（emoji + 位置frac + サイズ + 不透明度）
  }
  Course { ..., theme: WorldTheme }   // コースが世界を持つ
  ```
- **学年が上がるほど高い世界へ**という縦の物語を、テーマの並びで表現する（例）:

  | コース | 世界 | 空 | 主な飾り | ゴール |
  |---|---|---|---|---|
  | 小1 そうげん | 草原 | 水色→若草 | ☁️🌈🦋🌳🌷 | 🎈 |
  | 小2 うみべ | 海辺 | 空→海→砂 | ⛵🐠🌊🐚🦀 | 🏝️ |
  | 小3 もり | 森 | 空→深緑 | 🌲🦉🍄🦊🐿️ | 🏕️ |
  | 小4 やま | 山・雪 | 雪空→岩 | 🏔️❄️🦅🐐 | 🚩 |
  | 小6 うちゅう | 宇宙 | 夜→宇宙 | 🪐🌙☄️🛸⭐ | 🚀 |
  | 英検5級 きいろ | 太陽 | 黄グラデ | ☀️🌻🐥🌼 | 🏅 |

  → 学年（やコース）を進めるたびに「次の新しい世界」に着く設計。最終学年でようやく宇宙、にすると到達感が出る。
- **文字の可読性**: テーマで背景が明るくも暗くもなるため、白文字ラベルには軽い影（`legible()`）を付け、どのテーマ上でも読めるようにする。
- **拡張**: 季節背景画像（home-background 110種）をテーマの `skyStops` 代わりに敷く案も同じ仕組みで差し替え可能（§6）。
- **コスト最小化**: 飾りはまず絵文字で軽く置く。後でなかま/季節画像アセットに差し替えても `props` 配列を変えるだけ。

## 4. データ駆動の地図生成（自動再生成）

地図は **ステップ配列から座標を計算して描く**（位置はハードコードしない）。

- ノード座標 = 配列インデックスから算出（左右ジグザグ＋一定間隔で上方向）。
- 点線の小道は `[地上] + 全ノード + [ゴール方向]` を二次ベジェで結ぶ。
- **途中に新しいステップ（例: カメラ取り込み）が挿入されても、配列が変わるだけで道とノードが自動的に並び直る**（SwiftUI が再構築）。
  - → 「ステップの途中にカメラ取り込みステップができた時、ステップ選択画面は自動で再生成できるか？」の答えは **Yes**。
  - カメラ取り込みステップは、AI-OCR の手書き採点フロー（[ai-ocr-handwriting-research](./ai-ocr-handwriting-research-2026-06-27.md)）が生成・挿入する。生成タイミング（毎ステップ後 / N問ごと / 親設定）は別途決める（§8 Open）。

## 5. コース制（学年別 / 英検級別）

### 5.1 モデル
```
Course {
  id: String
  childName: String   // 子に見せるやさしい名前（"小1のたび" / "きいろのたび"）
  parentName: String  // 親に見せる正式名（"小1コース" / "英検5級コース"）
  emoji, accent, kind: .grade | .eiken
}
Step { ..., courseId: Course.id, kind: .word | .camera, ... }
```
- ステップは必ず1つのコースに属する。コースを切り替えると、その**コースのステップ地図に切り替わる**。
- 学年コース（小1〜）と英検コース（5級〜）を用意。英検級の対応は [eiken-level-mapping.md](./eiken-level-mapping.md) に従う。

### 5.2 設定・権限（決定事項）
- **既定: 親が設定・子は表示のみ**。子画面のコースチップは表示専用で切り替え不可。
- **親の設定画面に「子どもにもコースを選ばせる」トグル**を置く。
  - OFF（既定）: 子は親が設定した1コースのみ。チップは表示のみ。
  - ON: 親が**許可したコース群（allowedCourses）**の中で、子もチップから切り替え可能になる（チップが▼付きメニューになる）。
- 子に見せる場合も**級ラベルを直接出さない**（childName で表示）。

### 5.3 プロトタイプ上の対応
`StepMapPrototype.swift` では `childCanSwitch: Bool`（親トグル相当）で OFF/ON を切替。
- OFF → コースチップは表示のみ。
- ON → `allowedCourses` を並べた Menu に変化。

## 6. 既存資産の再利用
- **ホーム背景**（季節背景110種超）: 高度グラデの代わりに季節背景画像を敷き、その上に道を載せる構成も可（中央セーフゾーンに道）。実装時に検討。
- **アバター（avatar-dressup-gen）**: 「いまここ」足元のキャラ。
- **なかま（nakama-character-gen）**: 道中の点在キャラ。
- 角丸・カプセルバッジ・`.tapFeedback()`（バウンス＋Haptic）。

## 7. 演出（毎回出さない方針＝control-repeating-animated-ui）
- 起動時にキャラが前回クリア地点→「いまここ」へ歩く小演出は **初回/特定タイミングのみ**。毎起動では出さない。
- 「いまここ」ノードの拍動・きらきらは常時可（軽い）。タップでバウンス→アイリス遷移。
- クリア直後に戻ると、道が1区間伸びてスタンプが押される演出。
