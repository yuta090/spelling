1. 個人 or 法人 を選ぶ

まずどちらの名義で登録するかを決めます。年会費はどちらも $99/年（標準プログラム）です。
（$299/年の「Enterprise Program」は社内配布専用で App Store には出せません。今回は不要です。）

注意： 個人→法人の「移行」は手続きが必要で手間。会社名で出すと決めているなら、最初から法人で取るのが結局はラクです。

2. 事前準備（共通）

  • Apple Account（旧 Apple ID）を用意し、2ファクタ認証を必ずオンにする
  • Apple Account の姓・名は「本名」にする（ニックネーム・会社名は審査遅延の原因）
  • 支払い用のクレジットカード（個人登録は本人名義のカード必須）
  • 法人の場合：法人ドメインのメール（例 @skara.co.jp）、稼働している自社サイト、D‑U‑N‑S番号

3. 法人なら：D‑U‑N‑S番号を取る（無料）

D‑U‑N‑S（ダンズナンバー）は Dun & Bradstreet が発行する9桁の企業識別番号。法人登録の本人確認に必須です。

  1. まず Apple の確認ページで自社が既に D‑U‑N‑S を持っていないか検索する（持っていればそのまま使える）。
    Apple の D‑U‑N‑S 確認: developer.apple.com/enroll/duns-lookup/
  2. 無ければ同ページから無料で申請。法人名・住所・代表電話などが登記と一致している必要あり。
  3. 発行・反映まで数営業日〜2週間程度かかることがある（早めに着手）。
ポイント： 登録する法人名・住所は登記情報と完全一致させること。屋号・略称・支店名はNG。

4. 個人で登録する手順（Apple Developer アプリ推奨）

  1. iPhone/iPad に 「Apple Developer」アプリを App Store からインストール。
  2. 2ファクタ認証済みの Apple Account でサインイン。
  3. アプリ内の Account → Enroll を開く。
  4. Entity Type で「Individual / Sole Proprietor」を選ぶ。
  5. 氏名・住所などを本名・正確に入力（販売者名として公開される）。本人確認を求められたら画面の指示に従う。
  6. 規約（Apple Developer Program License Agreement）に同意。
  7. 年会費 $99（円表示）を支払う。アプリ経由は自動更新サブスクとして課金（更新は後でオフ可）。
  8. 確認メールが届く。反映まで時間差あり（24時間以内に来なければ Enrollment ID を控えてサポートへ）。
Web でやる場合：developer.apple.com/programs/enroll/ から同様に進められます。

5. 法人で登録する手順（Web）

  1. 契約締結権限を持つ人（代表・役員・権限委譲された社員）の Apple Account（2ファクタ認証済み）を用意。
  2. developer.apple.com/programs/enroll/ にサインインして開始。
  3. Entity Type で「Company / Organization」を選択。
  4. 法人名（登記名）・D‑U‑N‑S番号・法人ドメインのメール・自社サイトURLを入力。
  5. Apple による本人確認（D‑U‑N‑S照合＋締結権限の確認）。場合により公証済み書類や担当者への電話確認あり。
  6. 確認完了の連絡（メール）後に、規約同意＋$99 を支払い。
  7. 反映後、必要ならチームメンバーを追加できる。
遅延の主因： 法人名/住所が登記やD‑U‑N‑Sと不一致／サイトが未完成・ドメイン不一致／メールが法人ドメイン外。ここを揃えるとスムーズです。

6. 登録後：App Store Connect 初期設定

登録が完了したら App Store Connect で以下を行います（課金アプリに必須）。

  1. 契約・税・銀行（Agreements, Tax, and Banking）で 「Paid Applications（有料App契約）」を締結。 銀行口座・税情報（日本の税フォーム）・連絡先を登録。これが未完だとサブスクを売れません。
  2. Identifiers で App ID com.yuta090.SpellingTrainer を登録（Xcode 連携で自動作成されることも）。
  3. App Store Connect で新規 App を作成し、上記 Bundle ID を紐付け。名称・カテゴリ（教育）・年齢区分（Kids カテゴリにするなら子ども向け設定）等を入力。
  4. プライバシーポリシー・サポートURL・スクショ等のメタデータを用意（Kids＋サブスクは記載要件が多い。アプリ内ペイウォールの必須記載と整合させる）。

7. サブスク2商品＋7日トライアルを作る

App Store Connect → 対象App → 「収益化 / サブスクリプション」 で作成します。Product ID と価格はアプリ側と完全一致させてください。

  1. サブスクリプショングループを作成：名称 Parent Plan（保護者プラン）。
  2. グループ内に自動更新サブスクを2つ作成：
    表示名Product ID期間価格
    保護者プラン（月額）com.yuta090.SpellingTrainer.parentplan.monthly1ヶ月¥580
    保護者プラン（年額）com.yuta090.SpellingTrainer.parentplan.yearly1年¥4,800
  3. 各サブスクにIntroductory Offer（イントロオファー）= 無料・7日間（Free, 1 week）を追加。
  4. 各サブスクのローカライズ（日本語の表示名・説明）とレビュー用情報を入力。
  5. 審査用にサンドボックステスター（Users and Access → Sandbox）を作成し、実機サインアウト状態で購入テスト。
コードはもう準備済み： アプリ内の StoreManager はこの Product ID を参照しています。ローカルの Products.storekit で購入フローはテスト済みなので、ASC 側を同じ値で作れば本番接続できます（実URL・価格の整合だけ確認）。

8. （将来）親通知用 APNs キー

「子が学習したら親に通知」を実装する段階で必要になります（今は不要、登録後にいつでも作成可）。

  1. Certificates, Identifiers & Profiles → Keys → ＋ で新規キー作成、Apple Push Notifications service (APNs) を有効化。
  2. 生成される .p8 鍵ファイル（1回だけDL可）と Key ID・Team ID を安全に保管。
  3. これをサーバ（Supabase Edge Function 等）に登録して、親端末へプッシュ送信に使う。

9. ハマりどころ＆チェックリスト

  • Apple Account の2ファクタ認証が有効
  • 氏名/法人名が本名・登記名と一致（ニックネーム・屋号NG）
  • 個人登録は本人名義のクレカ
  • 法人はD‑U‑N‑S・法人ドメインのメール・稼働サイトを事前に用意
  • 登録後すぐPaid Applications 契約（銀行・税）を締結（サブスク販売の前提）
  • サブスクのProduct ID/価格をアプリと完全一致（…parentplan.monthly=¥580 /…parentplan.yearly=¥4,800）
  • 各サブスクに7日無料（Introductory Offer）を設定
  • Kids＋サブスクのメタデータ/プライバシー/保護者ゲート要件を満たす

料金の注意： 標準プログラムは個人・法人とも $99/年（登録画面で円建て表示・要確認。日本は近年 ¥12,000 前後）。$299 は Enterprise Program で App Store 配布には不要。
