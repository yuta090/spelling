前提ポジショニング: **「英検・中学英語の先取り × テストで覚える（検索練習）× 手書き × 親に可視化」**

## 0. なぜやるか（動機）
- 単語単体の練習だけでは「使える英語」として身につきにくい。文の中で語順・かたまりを体で覚えさせたい。
- ただし文を**全部手書き**は子どもに重く、採点も難しい。→ **選択式中心＋飽きない複数形式**で出す。
- このアプリの強み（手書き＋AI採点）は**産出形式（英作文・穴埋め手書き）**として温存し、入門は選択式で足場をかける。

## 1. 決定事項（ユーザー確認済み）
1. **形式はシステムが自動ミックス**（子に形式を選ばせない＝1画面1動作を維持）。
2. **学年の壁は絶対**：出題文の**内容語はすべて対象学年（=対象NGSLバンド/英検級セット）以内**。これは例外なし。
   - 範囲内であれば**未習語が文に混じってよい**（厳格な“既習のみ”にはしない）。
   - **わからない語は子がタップ→マーカー（ハイライト）→その語が学習導線に入る**（child ソースの単語として SRS box1 に登録）。これで「文づくり」と「単語学習」がつながる。
   - **登録後はいつものスペル練習に「ふくしゅう」として出題される**：最初は近いうちに、その後は SRS の間隔で再出題（box が上がるほど間隔が伸びる）。文づくりで出会った未習語が、自動でスペル練習の宿題になる。
3. **文データは2系統**：(a) ユーザーが事前に用意する厳選文（主） + (b) Tanaka Corpus から条件を満たすもの（使えるものがあれば採用）。
4. **着手は並べ替えのみ**。他形式は型だけ用意して空けておく（拡張可能な器にする）。

## 2. ドメイン全体像
スペル練習と並ぶ新ドメイン「**文づくり**」。内部的には全形式を 1 抽象 `SentenceExercise` に統一し、**出題形式（presentation/answer）だけ差し替える**。

共通の素は 3 つ：**正解英文 / 和訳 / 単語トークン列**。どの形式もここから生成。

```
SentenceItem(正解英文, 和訳, tokens[], gradeBand, contentLemmas[])
        │  ← 文バンク（同梱・学年タグ付き済み）
        ▼
ExerciseGenerator(format) ── SentenceExercise（提示＋正解＋採点方式）
        ▼
SessionComposer（自動ミックス・難度昇格・直前回避）── 出題列
        ▼
Grader（決定的 or AI）── 結果 → SRS 更新 / 未習語タップ→学習登録
```

## 3. データ：文バンクと「学年の壁」の実装

### 3.1 ビルド時タグ付け（前処理が本丸）
同梱 `wordbank.sqlite` の `level(word, ngsl_rank, band, dolch)` は **2,866語のみ**、dolch は約315語と疎。
→ 「全語が対象学年以内」を保証するには、**文ごとに前処理で学年を確定**する：

各候補文について、
1. 内容語に分解（lemma 化。冠詞・be 動詞等の機能語は学年判定から除外可、語順学習では tokens に含める）。
2. 各内容語の学年 = `level` から解決。
3. **全内容語が leveled 集合内 かつ band ≤ 上限** → 採用。`gradeBand = max(内容語の band)`。
4. 1語でも leveled 外 / 上限超 → **その band では不採用**（より上の band で再評価）。

結果を **同梱の派生テーブル `sentence_bank`**（静的リソース）に書き出す：
```
sentence_bank(id, en, ja, tokens_json, content_lemmas_json, grade_band, source)
  source = 'curated'（ユーザー厳選） | 'tanaka'（自動抽出）
```
→ アプリ実行時は `grade_band ≤ 子の対象band` で引くだけ（軽い・決定的）。**学年の壁はデータ生成時に保証**される。

### 3.2 2系統の取り込み
- **curated（主）**：ユーザーが用意する文（en/ja/狙う文法・対象band）。同じ前処理を通す（壁チェックは同様に適用、ただし「狙って入れた1語」は許容＝§4のストレッチ）。
- **tanaka（従）**：`examples`(61,570行) から、§3.1 を満たし**かつ子ども適切・短文**（語数しきい値、不適切語ブロックリスト）なものだけ自動抽出。

> ⚠️ 前処理パイプライン（lemma 化・学年解決・適切性フィルタ）は別タスク。まずは並べ替えコアを既存データで動かし、後で `sentence_bank` に差し替える。

## 4. 出題範囲ロジック（純粋・テスト対象）
入力：`SentenceItem`、子の `targetBand`、既習/SRS 状態。
- **ハードフィルタ（絶対）**：`item.gradeBand ≤ targetBand`。違反は出題しない。
- **未習許容**：範囲内なら未習語があってよい。提示時に未習語へ**タップ可能マーカー**を付ける（UI）。タップ → `EnrollUnknownWord`（child ソース・SRS box1）。
- **ストレッチ（curated）**：狙いの新出を1語含む文を優先的に混ぜる（学習効果）。
- 文・文法ポイント単位で **SRS を流用**（box で復習間隔）。

## 5. 出題形式カタログ（拡張可能 = 「選べる形」）
各形式は属性を持つプラグイン：`難度 / 入力(tap|handwrite) / 採点(deterministic|ai) / 認知(認識→産出)`。

| 形式 | 出し方 | 入力 | 採点 | 認知 | 着手 |
|---|---|---|---|---|---|
| **並べ替え** | 和訳→単語タイルを正しい順に | tap/drag | 決定的 | 認識+構成 | ★最初 |
| 穴埋め(選択) | 文に空所、選択肢から | tap | 決定的 | 認識 | 後 |
| 穴埋め(手書き) | 空所に語を手書き | handwrite | AI/OCR | 産出(部分) | 後 |
| 英作文「言いたいことを文に」 | 和訳→全文を作る | handwrite | AI VLM | 産出(最強) | 後 |
| (補)3択/正誤 | クイック確認 | tap | 決定的 | 認識 | 任意 |

同じ `SentenceItem` を形式違いで使い回せる（並べ替え→穴埋め→英作文の足場かけ）。

## 6. 飽きない仕組み（SessionComposer・純粋）
- **形式ミックス**＋**直前と同じ形式を避ける**。
- **習熟で形式を昇格**：新出文＝易しい形式（並べ替え）→ 定着＝難しい形式（英作文）。
- 子に形式選択をさせない（自動）。

## 7. 採点の抽象化
`Grader` プロトコルで形式から採点方式を隠す。
- 並べ替え・穴埋め(選択)：**決定的**（位置一致・部分点・差分提示）→ `SpellingSyncCore`。
- 穴埋め(手書き)・英作文：**AI VLM**（remote-grading-spec.md の出口を文へ拡張）。

## 8. 置き場所（プロジェクト規約）
- **`SpellingSyncCore`（純粋・TDD）**：`SentenceItem` / `SentenceExercise` / `ExerciseFormat` / 出題範囲フィルタ / 学年バンド選択 / `SessionComposer` / 決定的グレーダ（並べ替え→順次）/ 選択肢ダミー生成 / `EnrollUnknownWord` ロジック。
- **アプリ（薄く）**：タイル描画・ドラッグ・手書きキャプチャ・AI採点呼び出し・未習語マーカーのタップ UI。
- **データ**：`sentence_bank`（同梱 sqlite、学年タグ・トークン済み）。

## 9. UX（2人ユーザー）
- **子**：「文づくり」遊び。形式は自動ミックス。文法用語ゼロ。大きいタイル・ドラッグ・即発音/バウンス。未習語は色マーカー＋タップで「これ おぼえる？」。
- **親**：対象学年/英検級・文法範囲・有効形式・curated 文の追加 を管理コンソールで設定。
