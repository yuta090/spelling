## 1. 背景・哲学

- 例文は **Tanaka 抽出（大人題材）をやめ、AI で子の現在学年に合わせて生成**。
- 中心哲学＝**「楽しく文章をたっぷり浴び、数をこなすうちに自然と身につく」**（多読多聴 / comprehensible input）。
- **ただしスペル習得の代役にはしない**：浴びた量と「綴れた/選べた/直せた」は**別指標**で持つ。
- 子ども向けUI原則：子に専門用語/評価を見せない・1画面1動作・複雑さは親ゲートの奥。

## 2. 2トラック（必須／プール）＝ ステップに紐づく

| トラック | 中身 | 出方 |
|---|---|---|
| **必須（コア）** | **そのステップの登録語**（親driven・既存mastery） | ステップに来ると**最初にデフォルトで出る**。必ず1回やる |
| **プール（おたのしみ）** | **生成した例文**（tier準拠・ユーモア含む） | 必須クリア後・合間に**ランダム** |

### 自動フロー（子に設定は出さない）
1. ステップに来る → **必須問題が最初に出る**（メニューなし）。
2. **満点でクリア → そのステップは以後ランダム（自由）に自動切替**。
3. **次のステップ → またその必須問題がデフォルト**。
4. 「同じ問題を繰り返す」＝そのステップに戻ればよい／「新しい刺激」＝プールから。
5. まちがえた語の再出題は**既存の復習の仕組み（ReviewQueue / SRS）に委譲**。

## 3. 必須問題の契約（★レビューで確定した“約束ごと”）

ステップの登録語は親が自由に登録する（学校のテスト語など）。生成プールに合う文が無いこともある。だから必須は次を守る：

1. **任意の登録語は必ず問題化できる**（薄い仕分け役 `CoreProblemResolver` を1つ置く）：
   `綴りを変えないフレームに合致` → `音の似たおとりがあれば単語リスニング` → `直接スペル（その単語を打って答える）`。
   **最後の直接スペルは必ず成立**するので、巨大なマッチング機構は不要。**＝生成例文が1つも無くても必須は回る**（生成は“あれば嬉しい飾り”で前提条件ではない）。
2. **必須の答えは登録語“そのままの綴り”**。`apple` の必須で `apples` を答えさせない。
   → **必須に使うフレームは“単語の綴りを変えないもの”に限る**（例「This is an apple」）。変わる語は直接スペルへ。
3. **必須は“綴りを自分で出す形”**にする（打つ／将来は手書き）。**並べ替え（タイルを並べる）・選んで穴うめ（多肢選択）は綴りを書かせないので必須には使わない**（プール側ではよい）。手書き採点はモデル待ち、**「打って答える」直接スペルは今でも作れる**。
4. **ゆるい一致（見出し語/活用形）を必須判定に流用しない**（ヒント・プール露出にだけ使う）。← 最大リスクの封じ。
5. **年齢制約（band/文法/漢字）の効き先**：**生成文・周辺語・和訳・プール**に効かせる。**親登録語そのものは tier 例外**（親の自由）。i+1 は「その登録語を“許される未知語1語”として数える」だけ。
6. **必須の順番・クリア判定は、既存のステップ順＋満点ゲートに一本化**。生成コンテンツ側に別の順番（coreOrder）を持たない（二重管理回避）。`track=core` は静的な文に貼らず、実行時に「この登録語の必須」として扱う。
7. **必須完了の状態を保存**：`(ステップID, 単語構成のsignature)` で「このステップの必須は完了」を持つ。**単語を足す/入れ替えたら該当ステップの必須を再ロック**（古い満点が残らない）。完了キーは**単語の文字列でなく ID＋ステップ＋signature**。
8. **プールで出会った未知語は、勝手に子のステップへ足さない**。**復習にだけ積む**（満点ゲートと衝突させない）。

## 4. 生成コンテンツの2つの役割

- **フレーム（乗り物）**：登録語を**綴りを変えず**スロットに入れて必須/練習に使う。**最優先で充実させる**。
- **完成文（プール）**：日常会話・ユーモア・物語。**刺激・多読露出専用**（登録語を含むとは限らない・乗り物にしない）。
- **名前入り（personTemplate）＝プールの自然な一種（改訂2026-07-02）**：友達/自分の名前が出る文（倉庫3 `person_templates`／`docs/personalized-sentences-spec-2026-06-28.md`）。自分ごと化で定着に効くので**自然に混ぜる**。ただし**短さ・実用・ルビ・語彙/文法の壁は他の例文と同じ**（名前だからと長く・難しくしない）。役スロットは既存 fallbackEn/Ja を使い、名前未設定時は既定名にフォールバック。
- データ上は `kind: frameTemplate / plain / personTemplate` で分ける。出題プールと必須解決は**別物**として扱う。

## 5. 年齢別の中身（教育設計）

- **軸＝場面・機能（can-do）中心**。文法を見せる飾り文は作らない。文法は**天井（制約）**。
- **NGSL band は“ゆるい上限（rare語落とし）”に格下げ** ＋ **段階別「使ってよい語リスト」**で“子に既知か”を担保（cat/mom=band4 問題への対処）。
- **2本柱**：背骨＝役立つ日常会話 ／ スパイス＝**ユーモア（プール限定・全体10%）**。
- **フレーム＋入れ替え語**：型に練習語をはめる（綴りを変えない型を優先）。型ごとに「入れてよい品詞/意味カテゴリ」を持つ。
- **決まり文句（formulaic）**：普通の文型でない→**文法チェックのみ免除**（語彙・漢字・安全は免除しない・whitelist指定）。
- **ユーモアの歯止め**：おかしいのは“意味”だけ、**英語は100%正文**（壊れた英語・ナンセンス語禁止）。**目印（😄/ラベル）＋おかしな絵**で「わざと変」と伝える。段階aは現在形のみ。こわくない・きたなくない・いじわるでない。
- **漢字の壁（改訂2026-07-02）**：和訳の学年超過漢字は**捨てず「ルビ」で見せる**。境目は `KanjiLevelGate` の「1学年前」線を流用＝**gradeOf(漢字) ≤ 学年−1 は素／それ以外（当該学年以上・教育漢字外）はルビ**。＝既習は素・未習はふりがな。「捨てる」を廃したぶんカバー率が回復する。読みは**ビルド時生成＋検証**で `reading` として同梱（実行時OS辞書は保険）。全ひらがな化はしない（分かち書きで語境界は保つ）。→ §13。
- **例文の長さ（改訂2026-07-02）**：**全学年〈短い・具体・実用〉を保つ（目安3〜7語・コロケーション優先）。学年で長くしない**。長文は認知負荷が上がり覚えたい語が埋もれる（→§13 研究）。学年で変えるのは長さでなく〈語彙難度・文法天井・ルビ量〉。

### 4段階（題材・文法天井のグルーピング）＋ 学年別の壁
題材・文法天井は下の4段階でまとめる。**ただし〈語彙難度・ルビ量〉は学年(9段)で効かせる**（改訂2026-07-02）。**長さは全段階〈短い〉で固定**。
「漢字」列は旧＝捨てる閾値。**改訂後はルビ/素の境目**（gradeOf ≤ 学年−1 は素、それ以外はルビ）。

| 段階 | 学年 | 文法 上限 | 漢字（素/ルビの境目・学年別） | band | 長さ | 題材（加算式） |
|---|---|---|---|---|---|---|
| **a 入門** | 小1-2 | intro1 | 小1=全ルビ / 小2=小1漢字は素 | ゆるい上限 | 短い（3〜7語） | 下の6機能 |
| **b** | 小3-4 | intro2 | 小3=小2まで素 / 小4=小3まで素 | ゆるい | 短い | ＋さそう・たずねる・約束・お店で注文・天気・からだ |

あいさつ／きもち／すき・ほしい／へんじ／おれい・ごめん／たすけ。現在形・intro1・訳ひらがな・3〜5語。

### 著作権・画像
- **PD原作の“世界・場面”は借用可**（アリス/イソップ/グリム/人魚姫/ピノキオ/新美南吉/宮沢賢治…死後70年経過）。文は引用せず作り替える。

## 6. ヒント機能

- 解いている**最中**の助け（答え合わせ後の解説とは別）。未知語が必ず出るので必須。

## 10. データ・管理アーキテクチャ（スキーマv2）

- **正本authoring を分離**：`person_templates` の二枚使いをやめ、専用 authoring（行に **`kind: plain | personTemplate | frameTemplate`** 明示）。`sentence_bank.json` は派生のまま。複数エージェント衝突を減らすため**1行1レコード(JSONL)**を推奨。
- **最小 enum 軸**（自由タグ禁止＝崩壊防止）：`functionID` / `genre`(useful|humor|story) / `formulaic` / `frame`+`slot`+`allowedPOS`/`semanticClass` / `artHint` / `provenance` / `hint`。
- **ID分離**：安定 `sourceID`（教材）＋ 表層 `surfaceID`（文からUUIDv5）。英文修正で履歴が切れないよう `content_id_aliases.json` で移行。
- **`ContentPolicy`（Core）**：子の学年 →（tier / targetBand(ゆるい) / grammarCeiling / maxKanjiGrade / enabledGenres / i+1上限）。出題プール組立の入口で必ず適用。
- **built artifact に派生 `capabilities`**（canCloze / canSentenceAudio / canUseAsSpokenScript / needsAudioAsset…）。authoring に手書き `modes` は持たない。
- **生成を締める**：agy 出力を JSON/JSONL に固定 → `content-lint`（enum/重複/近似重複/見える語/漢字/formulaic whitelist/genre/**機能カバレッジ＋件数クォータ**）→ Claude は機械通過後の差分だけレビュー。**10k一括せず少量同梱**。
- **多エージェント協調**：repo 管理の **`agent-claims/`**（owner/files/intent/branch/expiresAt=TTL）＋ preflight 重複警告。
- **移行**：既存47件を v2 へ手移行（v1配列 / v2 envelope の dual decode で安全に）。CLI は **却下が1件でもあれば書き出さない**を既定（`--fail-on-rejects`、例外時のみ `--allow-partial`）＝スキル記述と一致させる。

## 13. 改訂2026-07-02 の決定と根拠

### 13.1 例文の長さ＝全学年で短いまま（学年で増やさない）
- **理由**：文の長さは読みの難しさの主要因で、長いほど作業記憶に負荷（認知負荷）→過負荷は学習効果を下げる。短文の目安3〜6語。語を覚える目的では**覚えたい語を主役にできる短文**が有利。
- **かたまり（コロケーション）で覚える**方が定着・実用に効く（`take medicine` 等）。子は**短く具体的な文＋複数文脈**（絵・音・やりとり）で定着。
- **結論**：長さは学年で変えない。学年で変えるのは〈語彙難度・文法天井・ルビ量〉。副次効果として短文固定は**生成しやすく在庫（カバー率）が増える**。

### 13.2 文法は「目標」でなく「天井」（綴りヒント例文では）
- 綴りヒント例文の仕事は**その単語の意味・使い方を見せる**こと。文法は **parse できる上限（天井）** として使い、**その学年の新出文法を必ず入れる目標にはしない**。
- **文法の目標的練習は「ことばパズル（ぶんづくり/あなうめ）」側が担当**（役割分担）。将来、綴り側に文法教授モードを足すならそのとき目標文法を入れる。

### 13.3 和訳のふりがな（ルビ）＝「捨てる」を置換
- 全ひらがなは**逆に読みにくい**（同音異義＋語境界が消える）。教科書も小2途中まで**分かち書き**で補う。漢字は語境界の視覚手がかり＝**既習漢字は残す方が読みやすい**（目安 漢字3:かな7）。
- **ふりがなは「音」を解決するが「意味」は解決しない**（読解＝語彙＋構文＋背景知識）。→ ルビを付けても**低学年は和訳語彙をやさしく保つ**（band/語リストで担保・ルビとは別の壁）。
- **ルール**：`gradeOf(漢字) ≤ 学年−1`＝素／当該学年以上・教育漢字外＝ルビ。学年内の**習う順（ステップ）は全国共通でないため使わず、当該学年の漢字は一律ルビ**（4月/3月を気にしない）。読みはビルド時生成＋検証で同梱。

### 13.4 粒度＝学年別（9段）
- 題材・文法天井は4段階でまとめるが、**語彙難度とルビ量は学年で効かせる**（小1-April と小2-March は別物）。`KanjiLevelGate.maxGrade(forSchoolGrade:)` は既に学年別なのでルビ側は流用できる。

### 13.5 名前入り（personTemplate）をプールに自然に含める
- 自分ごと化は定着に効くので混ぜる。ただし**短さ・実用・ルビ・語彙/文法の壁は他と同一**。

### 13.6 英語語彙の基準＝CEFR-J（採用候補）
- NGSL（英語頻度）に代え/加えて **CEFR-J Wordlist**（日本人学習者基準・A1.1〜B2）を採用候補。機械可読で商用可の **Open Language Profiles（olp-en-cefrj）CSV 版**が App 同梱向き。出典表記は保護者メニュー→設定に掲載（Tanaka/EJDict と同運用）。
