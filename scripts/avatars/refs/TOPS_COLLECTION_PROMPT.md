# agy（Antigravity）向け：トップス参考画像 収集プロンプト

子ども向けiPadアプリ「着せ替えアバター」のトップス生成に使う**参考画像（=服の“形・構造”の手本）**を、
ファッション系統(kei)別に収集してもらうための指示。下の「===」ブロックをそのまま agy に渡す。

> 注意：参考画像は **形の手本**として使うだけ。最終アートは別途オリジナルのチビ絵で描き起こす。
> ブランドのロゴ/プリント文字が写った画像は避ける（アプリは文字・ロゴ不可、子ども向け）。

あなたはファッションリサーチャーです。子ども向けアプリのアバター服デザインの**参考画像**を集めてください。
収集対象リストは下記。各アイテムにつき **2〜3枚** の参考画像を探して保存します。

## 各画像の要件（重要）
- **正面・平置き(フラットレイ) か 商品写真**。服の構造（襟・袖・前開き/ジップ・丈）がはっきり分かるもの。後ろ姿・斜め・暗い・服が小さい写真はNG。
- **2026年の今っぽいトレンド**を意識（オーバーサイズ/レトロスポーツ/Y2K/韓国系/ナチュラル等）。ただし**子どもが着て可愛い・上品(modest)**な範囲。露出が高い・大人っぽすぎるものはNG。
- **ブランドのロゴ・大きなプリント文字が写っていないもの**を優先（無地・アイコン・チェック・ボーダー・カラーブロックはOK）。
- 1枚は「全体の形」が分かる引き、可能なら1枚は「襟元/袖口」が分かる寄りもあると良い。

## 保存ルール
- 保存先: `scripts/avatars/refs/<kei>/`（kei フォルダは下記）。
- ファイル名: `<id>__1.jpg` `<id>__2.jpg` …（id は下記リストのもの）。
- 各 kei フォルダに `SOURCES.md` を作り、画像ごとに「ファイル名 / 取得元URL / ライセンスや出典の注記」を1行で記録。
- **ライセンス配慮**: 商用利用や再配布が不明な画像は「形の参考のみ・再配布しない」と注記。明らかに著作権の強い作品（イラストレーター個人作品等）は避け、商品写真/素材サイト/自由ライセンスを優先。

## 集まった後（こちら側の使い方）
1. `tops.csv` の `ref` 列のパスに参考が置かれていることを確認。
2. 生成時、codex の cwd(`out/`) に該当 ref をコピーして「この参考は**服の形とディテールだけ**真似る。レンダリングは我々のチビ・セル塗りで、base の腕ジオメトリ＆首クリップ(y<632)ルールに従う」と指示。
3. `_top_finish.py` で自動QA→目視→出荷。`GENERATION_LOG.md` にラウンド経過を追記。
